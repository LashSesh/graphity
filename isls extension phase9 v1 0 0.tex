\begin{abstract}
\noindent
This document specifies the \ISLS{} Studio: a single-page web
application served by the Gateway (C19) that provides a unified
operator interface for the entire substrate. No new Rust crate is
needed --- the Studio is a static HTML/JS/CSS bundle served by the
existing Axum server, communicating via the existing REST and
WebSocket APIs.

The Studio has seven views: Dashboard (system health at a glance),
Forge (describe what you want, watch it build), Explorer (browse
crystals, manifests, patterns), Monitor (live engine metrics and
events), Foundry (project fabrication with compile feedback), Oracle
(autonomy metrics, pattern memory, budget), and Constitution (Genesis
Crystal, ADAMANT status).

Design principles: dark theme, keyboard-first, minimal clicks,
real-time updates via WebSocket, zero external dependencies (no npm,
no React, no build step --- pure vanilla HTML/JS/CSS served as a
single file from the binary).

\medskip\noindent\textbf{Status:} Implementation Specification.\\
\textbf{Parent:} ISLS v1.0.0 + Extensions Phase 1--8.\\
\textbf{Depends on:} C19 (Gateway REST + WebSocket), all prior crates
  (exposed via Gateway).\\
\textbf{Implementation:} Single HTML file embedded in the
  \texttt{isls-gateway} crate as a static asset. No new crate.
\end{abstract}

\part{Architecture}\label{part:arch}

\section{Why a Web UI}\label{sec:why}

The Gateway (C19) already serves REST and WebSocket endpoints. The
HTML report already demonstrates that the system can render rich
output. The Studio extends this to an interactive, real-time
interface without adding any new backend code --- it consumes the
existing API.

\begin{itemize}[nosep]
  \item No new Rust crate. The Studio is a static asset in
        \texttt{isls-gateway}.
  \item No build toolchain (no npm, no webpack, no React). Pure
        HTML + vanilla JS + CSS. One file.
  \item Served at \texttt{http://localhost:8420/studio} when
        the gateway is running.
  \item Dark theme matching the existing HTML report aesthetic.
  \item Real-time via WebSocket (\texttt{ws://localhost:8420/events}).
  \item Keyboard shortcuts for every action.
\end{itemize}

\section{Technology}\label{sec:tech}

\begin{lstlisting}[title={No build step. One file.}]
isls-gateway/
  src/
    static/
      studio.html     <-- THE ENTIRE STUDIO
                          HTML + CSS + JS in one file
                          Embedded as include_str! in Axum
                          Served at GET /studio
\end{lstlisting}

The Studio uses:
\begin{itemize}[nosep]
  \item Vanilla JavaScript (ES2020+, no framework)
  \item CSS Grid + Flexbox for layout
  \item WebSocket for real-time events
  \item \texttt{fetch()} for REST API calls
  \item \texttt{<canvas>} for charts (no chart library ---
        hand-drawn with Canvas API)
  \item CSS custom properties for theming
  \item \texttt{localStorage} for user preferences (panel sizes,
        last view)
\end{itemize}

\section{WebSocket Integration}\label{sec:ws}

\begin{lstlisting}[title={Studio WebSocket Client}]
const ws = new WebSocket('ws://localhost:8420/events');

ws.onmessage = (event) => {
    const msg = JSON.parse(event.data);

    switch (msg.type) {
        case 'tick':
            updateMonitorTick(msg.data);
            break;
        case 'crystal':
            updateDashboardCrystal(msg.data);
            addExplorerCrystal(msg.data);
            break;
        case 'gate':
            updateMonitorGate(msg.data);
            break;
        case 'alert':
            showAlert(msg.data);
            break;
        case 'forge_progress':
            updateForgeProgress(msg.data);
            break;
        case 'foundry_progress':
            updateFoundryProgress(msg.data);
            break;
        case 'metric':
            updateMonitorChart(msg.data);
            break;
    }
};

// Subscribe to relevant events based on current view
function switchView(view) {
    ws.send(JSON.stringify({
        subscribe: getEventsForView(view)
    }));
}
\end{lstlisting}

\part{Implementation Notes}\label{part:implnotes}

\section{Single-File Architecture}\label{sec:singlefile}

The entire Studio is one HTML file with inline CSS and JS. This is
deliberate:
\begin{itemize}[nosep]
  \item No build step.
  \item No node\_modules.
  \item No framework dependency.
  \item Embedded in the Rust binary via \texttt{include\_str!}.
  \item Works offline (once loaded, only needs the API).
  \item Estimated size: 30--50 KB (fits in a single TCP packet after
        gzip).
\end{itemize}

\section{Canvas Charts}\label{sec:charts}

Line charts are drawn directly on \texttt{<canvas>} with a minimal
charting function (~100 lines of JS). No external chart library.

\begin{lstlisting}[title={Minimal Canvas Chart}]
function drawLineChart(canvas, data, opts) {
    const ctx = canvas.getContext('2d');
    const { width, height } = canvas;
    const max = Math.max(...data);
    const min = Math.min(...data);
    const range = max - min || 1;

    ctx.clearRect(0, 0, width, height);
    ctx.strokeStyle = opts.color || '#3b82f6';
    ctx.lineWidth = 2;
    ctx.beginPath();
    data.forEach((v, i) => {
        const x = (i / (data.length - 1)) * width;
        const y = height - ((v - min) / range) * height;
        i === 0 ? ctx.moveTo(x, y) : ctx.lineTo(x, y);
    });
    ctx.stroke();
}
\end{lstlisting}
