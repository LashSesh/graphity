\section{Executive Summary}
% ══════════════════════════════════════════════════════════════════════════════

The current Studio (\texttt{studio.html}) has 9 tabs, dozens of 
buttons, and the aesthetic of a rocket cockpit. It was built 
incrementally --- each D-stage added a tab, nothing was removed. 
It is functional and ugly.

S1 replaces it entirely. One HTML file. Two modes. No tabs, no 
sidebar menus, no settings screens. The design reference is 
Claude.ai: a chat interface where everything happens through 
conversation.

\subsection{Design Requirements (from operator interview)}

\begin{itemize}[nosep]
\item \textbf{Reference:} Claude.ai / ChatGPT feel.
\item \textbf{Pain point:} Too many buttons and options.
\item \textbf{Theme:} Dark mode, always. Not pure black --- 
      dark gray (\texttt{\#0d1117}). Text not pure white --- 
      dampened (\texttt{\#e6edf3}). Accent: warm orange 
      (\texttt{\#d4871b}). Optimized for long night sessions.
\item \textbf{Devices:} Laptop (primary) + Mobile (ideas on the go).
\item \textbf{Sessions:} Long evening/night sessions.
\item \textbf{Transparency:} Show everything that happens --- 
      but in the conversation flow, not in separate panels.
\item \textbf{Responsive:} Must work on phone (chat is naturally 
      mobile-friendly).
\end{itemize}

\subsection{The Two Modes}

\begin{description}
\item[\ding{91} Erschaffen] The architect chat. Describe $\to$ 
  refine $\to$ forge $\to$ result. Everything in one conversation 
  stream. Readiness indicator. Entity cards inline. Forge progress 
  as chat messages. Result as expandable file tree.
\item[\ding{72} Entdecken] The feeding engine. Search GitHub for 
  patterns. Scrape repos with one click. X-ray any repo's topology. 
  Upload keyword lists for overnight mass-scraping. Norm genealogy 
  and gap analysis. Everything in one conversation stream.
\end{description}

Mode switch: a 40px vertical strip on the left with two icons. 
No text labels. Active mode has an accent dot. That is the entire 
navigation.

\section{Visual Design}
\label{sec:design}
% ══════════════════════════════════════════════════════════════════════════════

\subsection{Color Palette}

\begin{longtable}{lll}
\toprule
\textbf{Role} & \textbf{Color} & \textbf{Hex} \\
\midrule
Background (primary)   & Near-black        & \texttt{\#0d1117} \\
Background (cards)     & Dark gray         & \texttt{\#161b22} \\
Background (input)     & Slightly lighter  & \texttt{\#1c2128} \\
Text (primary)         & Dampened white    & \texttt{\#e6edf3} \\
Text (secondary)       & Medium gray       & \texttt{\#8b949e} \\
Text (dim)             & Dark gray         & \texttt{\#484f58} \\
Accent (primary)       & Warm orange       & \texttt{\#d4871b} \\
Accent (success)       & Muted green       & \texttt{\#3fb950} \\
Accent (error)         & Muted red         & \texttt{\#f85149} \\
Accent (info)          & Muted blue        & \texttt{\#58a6ff} \\
Border                 & Subtle            & \texttt{\#30363d} \\
\bottomrule
\end{longtable}

\textbf{Zero pure black. Zero pure white.} Every color is 
dampened. This reduces eye strain during 3+ hour night sessions.

\subsection{Typography}

\begin{itemize}[nosep]
\item Body: \texttt{-apple-system, BlinkMacSystemFont, "Segoe UI", 
      sans-serif} at 15px. Line-height 1.6.
\item Code: \texttt{"JetBrains Mono", "Fira Code", "Consolas", 
      monospace} at 13px.
\item No bold headings. Weight differences via size and color, 
      not boldness. Gentle hierarchy.
\end{itemize}

\subsection{Layout}

\begin{lstlisting}[style=bash]
+--+--------------------------------------------------+
|  |                                                    |
|  |                                                    |
|  |          Conversation Stream                       |
|  |          (scrollable, full height)                 |
|  |                                                    |
|  |                                                    |
|  |                                                    |
|  +--------------------------------------------------+
|  | [input field                          ] [O] [>]   |
+--+--------------------------------------------------+
 ^
 |
 Mode strip (40px):
   icon 1 = Erschaffen
   icon 2 = Entdecken
\end{lstlisting}

The mode strip is the ONLY persistent UI element. Everything 
else is conversation content.

The \texttt{[O]} next to the input is the readiness ring 
(Erschaffen mode only) --- a tiny circle that fills with the 
accent color as completeness increases. The \texttt{[>]} is 
the send button (or Enter key).

\subsection{Mobile Layout}

Same structure. Mode strip becomes a horizontal bar at the top 
(2 icons, centered, 32px tall). Conversation fills the screen. 
Input fixed at bottom. Swipe gesture: left/right switches mode.

\section{Mode 1: Erschaffen (\ding{91})}
\label{sec:erschaffen}
% ══════════════════════════════════════════════════════════════════════════════

\subsection{Empty State}

Dark background. Centered in the middle of the screen, large 
placeholder text: ``Was soll entstehen?'' in secondary color. 
Below it: three ghost suggestions in dim text:

\begin{itemize}[nosep]
\item ``Trading Journal f\"ur Crypto-Portfolios''
\item ``CLI Tool zum Konvertieren von CSV nach JSON''
\item ``REST API f\"ur Buchungsverwaltung''
\end{itemize}

Clicking a suggestion fills the input field. This is the only 
moment where clickable text exists. After the first message, 
the suggestions disappear forever.

\subsection{Conversation Flow}

\subsubsection{User Messages}

Right-aligned, accent background (\texttt{\#1c2128} with 
orange-tinted left border), rounded corners. The user's text 
in primary color.

\subsubsection{System Messages}

Left-aligned, card background (\texttt{\#161b22}), rounded 
corners. Three subtypes:

\paragraph{Response Messages.} The system's conversational reply. 
Normal text. May contain inline entity cards.

\paragraph{Entity Cards.} Compact inline cards showing entity 
name + field count. Tap/click to expand and see all fields with 
types. Collapsible. Background slightly distinct 
(\texttt{\#1c2128}).

\begin{lstlisting}[style=bash]
+---------------------------------------------+
| Trade (9 fields)                        [v]  |
+---------------------------------------------+
  Expanded:
  | pair         String                        |
  | side         String (buy/sell)             |
  | entry_price  f64                           |
  | exit_price   f64                           |
  | quantity     f64                           |
  | timestamp    String                        |
  | notes        String (nullable)             |
  | -> Portfolio (FK)                          |
+---------------------------------------------+
\end{lstlisting}

\paragraph{Process Messages.} Forge progress, scraping updates, 
norm events. Smaller font (13px), monospace for technical parts, 
left-aligned, no card background --- just inline text with 
subtle left border (accent color, 2px).

\begin{lstlisting}[style=bash]
  | S4 Solve: 21/50 nodes
  |   structural  models/trade.rs
  |   swarm(4)    database/trade_queries.rs
  |     T1 D=0.72  T2 D=0.68  T3 D=0.61  T4 D=0.70
  |     Omega=0.82  Dtotal=0.55 -> Mandorla: T1
  |   structural  services/trade.rs
\end{lstlisting}

\subsection{Readiness Ring}

A small circle (16px diameter) to the left of the send button. 
Fills clockwise with the accent color as readiness score increases 
(0--100\%). At 100\%: solid accent circle with a subtle pulse 
animation (CSS \texttt{@keyframes}, 2s period).

Tapping the ring shows a tooltip/popup with the criteria checklist:

\begin{lstlisting}[style=bash]
  Readiness: 85%
  [x] Entities vorhanden (3)
  [x] Felder definiert
  [x] App-Name: crypto-journal
  [x] FK-Targets valide
  [x] Beschreibung vorhanden (+10)
  [ ] Alle Typen definiert (+15)
  [x] Beziehungen vorhanden (+10)
  [x] 3+ Entities (+15)
  
  Tipp: PerformanceMetric.sharpe_ratio hat keinen Typ.
\end{lstlisting}

\subsection{Forge Trigger}

Three ways to trigger forge:

\begin{enumerate}[nosep]
\item Type ``forge'' or ``materialisieren'' or ``bauen'' in the 
      input field.
\item Press the readiness ring when it is full (100\%).
\item The system suggests it: after a message where readiness 
      reaches 100\%, the response ends with: 
      ``Bereit zum Materialisieren. Tippe \emph{forge} oder 
      dr\"uck den Ring.''
\end{enumerate}

No dedicated forge button. The ring IS the button. The 
conversation IS the interface.

\subsection{Forge Progress in Chat}

When forge starts, process messages stream into the conversation:

\begin{lstlisting}[style=bash]
  SYSTEM: Materialisierung gestartet.
  |
  | HDAG: 42 Nodes, 67 Edges
  | Infrastructure: Web-API + Database + Auth (kein Frontend)
  |
  | [1/42]  .gitignore
  | [2/42]  Cargo.toml
  | ...
  | [18/42] database/trade_queries.rs  [swarm]
  |   4 Thronengel konvergieren...
  |   T1=0.72  T2=0.68  T3=0.61  T4=0.70
  |   Dtotal=0.55 -> Mandorla: T1
  | ...
  | [42/42] tests/api_tests.rs
  |
  | cargo check: PASSED (0 errors)
  |
  SYSTEM: Fertig. 42 Dateien, 1890 LOC, 4m 12s.
  Ergebnis: ./output/crypto-journal/
  
  [Dateien anzeigen v]
\end{lstlisting}

``Dateien anzeigen'' is an expandable section (like an entity 
card) showing the generated file tree. Tap a file to see its 
content inline (syntax-highlighted, read-only).

\subsection{Session Persistence}

Sessions are listed in the mode strip area when expanded (tap 
the Erschaffen icon to see recent sessions, like Claude.ai's 
conversation list). Each session shows: app name, entity count, 
date, forge status. Tap to resume.

\section{Backend: New API Endpoints}
\label{sec:api}
% ══════════════════════════════════════════════════════════════════════════════

\subsection{Entdecken-Mode Endpoints}

\begin{longtable}{llp{6cm}}
\toprule
\textbf{Method} & \textbf{Path} & \textbf{Purpose} \\
\midrule
POST & \texttt{/api/discover/search}
  & GitHub keyword search. Body: \texttt{\{query, max\_results\}}.
    Returns repo list with metadata. \\
POST & \texttt{/api/discover/xray}
  & R\"ontgenblick. Body: \texttt{\{url\}}. Shallow-clones, scans, 
    returns topology summary without feeding norms. \\
POST & \texttt{/api/discover/scrape}
  & Scrape single repo into norms. Body: \texttt{\{url\}} or 
    \texttt{\{path\}}. Returns artifacts + candidate updates. \\
POST & \texttt{/api/discover/mass-scrape}
  & Mass scraping. Body: \texttt{\{keywords: [list], 
    results\_per\_keyword: 3\}}. Runs in background. Progress 
    via WebSocket. \\
GET & \texttt{/api/discover/mass-scrape/\{id\}/status}
  & Poll mass-scrape progress. \\
POST & \texttt{/api/discover/upload-keywords}
  & Upload keyword file. Body: multipart form with \texttt{.txt} 
    file. Parses keywords, starts mass-scrape. \\
GET & \texttt{/api/discover/gaps}
  & Norm gap analysis. Returns weak areas + search suggestions. \\
GET & \texttt{/api/discover/genealogy/\{norm\_id\}}
  & Norm genealogy. Returns observation history, domains, 
    promotion timeline. \\
GET & \texttt{/api/discover/similarity}
  & Repo similarity search. Query param: \texttt{?domain=fsr-engine}.
    Returns ranked list of similar scraped repos. \\
\bottomrule
\end{longtable}

\subsection{GitHub Search Integration}

Uses the GitHub Search API (unauthenticated: 10 requests/minute, 
authenticated with token: 30/minute). Search query constructed as:

\begin{lstlisting}[style=bash]
GET https://api.github.com/search/repositories
    ?q={keywords}+language:rust
    &sort=stars
    &order=desc
    &per_page={max_results}
\end{lstlisting}

For non-Rust searches (user types ``python flask''), omit the 
\texttt{language:rust} filter.

\subsection{Mass-Scrape Background Job}

Mass scraping runs as a background \texttt{tokio::spawn} task. 
Progress events published via WebSocket:

\begin{lstlisting}[style=bash]
{ "type": "discover:keyword", "index": 3, "total": 47,
  "keyword": "rust database orm" }
{ "type": "discover:repo", "keyword": "rust database orm",
  "repo": "launchbadge/sqlx", "artifacts": 456 }
{ "type": "discover:complete", "repos_scraped": 141,
  "repos_failed": 12, "new_candidates": 23,
  "new_norms": 4 }
\end{lstlisting}

Rate limiting: 2-second delay between GitHub API calls to stay 
within rate limits. Scraping itself (clone + parse) has no rate 
limit.

\subsection{W1: New studio.html}

Complete rewrite of \texttt{isls-gateway/src/static/studio.html}. 
Delete the old file. Start from blank. Single HTML file, inline 
CSS and JS, zero external dependencies.

Structure:
\begin{itemize}[nosep]

\item Conversation container (scrollable, full height).
\item Input area (fixed bottom: textarea + readiness ring + send).
\item Message renderer (user messages, system messages, entity

\subsubsection{Message Types (JS rendering)}

\begin{lstlisting}[style=rust]
// Message types the frontend renders:
enum MessageType {
    UserMessage,      // right-aligned, accent border
    SystemResponse,   // left, card background

\subsection{W3: Entdecken-Mode Backend}

New API endpoints in \texttt{isls-gateway}:

\subsubsection{New Files}

\toprule
\textbf{File} & \textbf{Purpose} \\
\midrule

\subsubsection{GitHub Search}

Uses \texttt{reqwest} (already in workspace) to call GitHub API. 
Parses response JSON, extracts repo name, stars, description, 
language. Returns as JSON to frontend.

\subsubsection{X-Ray}

Calls D5's \texttt{cmd\_scrape} logic but without calling 
\texttt{observe\_and\_learn()} --- read-only topology extraction.
Returns \texttt{CodeTopology} as JSON.

\subsubsection{Mass-Scrape}

Background job that iterates through keywords:
\begin{enumerate}[nosep]
\item For each keyword: GitHub search (top $N$ results).
\item For each result: shallow clone + Barbara + bridge + 
      \texttt{observe\_and\_learn()}.
\item Publish WebSocket progress events.
\item 2-second delay between GitHub API calls.
\item On completion: summarize new candidates and auto-norms.
\end{enumerate}

\subsubsection{Norm Gaps}

Analyze the norm catalog for areas with low coverage:
\begin{enumerate}[nosep]
\item List all layer combinations present in norms.
\item Identify common software patterns NOT covered 
      (hardcoded list of 20+ common patterns: caching, 
      queuing, rate limiting, circuit breaker, etc.).
\item For each gap: suggest search keywords.
\end{enumerate}

\end{enumerate}
